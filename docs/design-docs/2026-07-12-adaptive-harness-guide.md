# 適応的な開発ハーネスをスキルで補助する

状態: accepted

日付: 2026-07-12

決定者: リポジトリ利用者

見直し: スキルの実運用で過剰な手順強制、文書漏れ、または判断待ちの不足が確認された時
関連: [作業ルール](../../AGENTS.md)、[アーキテクチャ](../../ARCHITECTURE.md)、[実行計画](../exec-plans/completed/2026-07-12-adaptive-harness-guide.md)、[スキル](../../.agents/skills/harness-aidlc-guide/SKILL.md)

## 背景

このテンプレートは、仕様、設計判断、実行計画、品質証拠をコードとともに残すハーネスを提供する。新規アプリの立ち上げと継続的な機能追加では必要な工程と深さが異なるため、全変更に同じ工程を強制すると不要な文書や判断待ちが生じる。

AI-DLCの「AIが経路を提案し、人間が重要な判断と成果を検証する」考え方を取り入れつつ、ツールや特定の技術スタックには依存しない補助を必要とする。

## 決定

リポジトリの正本と検証を開発ハーネスとして維持し、その上で `harness-aidlc-guide` スキルを提供する。スキルは依頼の意図、既存コンテキスト、影響、リスクに応じて、Inception、Construction、Operations の各段階と成果物の深さを提案する。

スキルは固定ワークフローとして全段階を要求しない。製品範囲、永続的な技術判断、外部影響、セキュリティ、不可逆な操作は人間の明示確認を求める。ゴール機能は明示依頼時の短期進捗管理に限り、永続的な知識はリポジトリ内の正本に残す。

## 検討した選択肢

| 選択肢 | 採用しない／採用する理由 |
| --- | --- |
| 全変更を同じSDLC手順で実行する | 不採用。小さな修正や限定的な品質改善にも過剰な儀式を課す。 |
| 文書と`AGENTS.md`だけで運用する | 不採用。正本は維持するが、依頼に応じて経路・深さ・承認点を選ぶ補助が不足する。 |
| 適応的な開発ガイドスキルを追加する | 採用。ハーネスを正本として尊重しながら、AIが経路を提案し、人間の判断を保護できる。 |

## 影響

- 新規アプリ、機能追加、修正、運用作業に共通の入口を提供する。
- スキルを使うエージェントは、文書作成の有無と深さを根拠付きで説明する。
- リポジトリの文書、テスト、設定、CIが継続して永続的な正本となる。
- スキルの判断品質は実利用で検証し、過不足はスキル本文または関連文書へ反映する必要がある。

## 守り方

- スキル本文で、変更前の正本確認、重要判断での明示承認、索引・証拠の同期を指示する。
- `quick_validate.py`でスキル構造を検証する。
- Markdownリンクの存在確認と`git diff --check`を実施する。
