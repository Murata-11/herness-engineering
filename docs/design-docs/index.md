# 設計判断

## AIの手順

1. 既存の仕様・判断・実装を確認し、[設計判断テンプレート](../templates/decision-record.md)を`YYYY-MM-DD-<decision-name>.md`（英小文字kebab-case）へ複製する。
2. 1文書に1判断として、背景、決定、選択肢、影響、守り方を埋める。
3. この索引へ文書名・状態・用途を追加し、仕様・計画・実装を相互リンクする。守り方をテスト・CI・手順へリンクする。

`proposed`は検討中、`accepted`は実装上の制約、`superseded`は後継判断へリンクする。`accepted`と矛盾する変更は、上書きせず後継の`proposed`として記録する。
