# 実行計画

複数の変更・判断・検証が必要な作業を管理します。

## 文書一覧

| 文書 | 状態 | 用途 |
| --- | --- | --- |
| [進行中の計画](active/index.md) | 進行中 | 実施中またはレビュー中の計画 |
| [完了済みの計画](completed/index.md) | 保管 | 完了した計画と結果 |
| [技術的負債トラッカー](tech-debt-tracker.md) | 継続管理 | 既知の改善事項を優先付ける |

## AIの手順

1. 実装前に`active/`へ計画を作り、仕様・判断に基づく範囲、非目標、検証を記入する。
2. マイルストーンごとに根拠と検証結果を進捗へ、方針変更を決定ログへ追記する。
3. 全検証が成功したら結果・フォローアップを記録して`completed/`へ移動し、両索引を更新する。未実施・失敗なら`active/`に残す。

今回実施しない改善事項は、理由・影響・次の判断時期・関連リンクとともに技術的負債トラッカーへ追加します。
