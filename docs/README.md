# ドキュメント

コードだけでは表しにくい意図・判断・実行状況の正本です。

## AIを含む作業手順

1. 関連文書・実装を読んで正本を確認する。
2. 対象の`index.md`に従い、テンプレートを複製してプレースホルダー、状態、関連リンクを埋める。
3. 同じ変更で索引を更新し、関連文書と相互リンクする。
4. リンクと実装・テストの対応を確認し、`git diff --check`を実行する。未実施の検証は理由を残す。

根拠のある作成・更新・進捗記録はAIが自律的に行う。スコープ、承認済み判断、外部への約束を新たに固定する場合は、推測せず未決事項と選択肢を記録して判断を求める。

## 文書群

- [product-specs](product-specs/index.md): 利用者の行動、スコープ、受け入れ条件
- [design-docs](design-docs/index.md): 継続して参照する設計判断
- [exec-plans](exec-plans/index.md): 複雑な作業の計画、進捗、決定、フォローアップ
- [QUALITY.md](QUALITY.md): 品質条件、検証の証拠、既知課題
- [templates](templates/README.md): 新しい文書のコピー元

所有者・見直し契機を記載し、実装・設定と重なる値は複製せず正本へリンクします。
